# 한국형 생활 계산기 포털 — 설계도

> 목표: **비용 0(도메인값 제외)으로 시장 수요를 검증**하고, 검증된 도구만 골라
> iOS 앱으로 승격시켜 애플 개발자 계정 $99를 정당화한다.
> 코어 자산: 음력 엔진(KoreanLunarKit ≈ KASI/KARI 기준), 경조사 룰테이블.

---

## 0. 전략 한 줄

**웹 = 미끼(트래픽·검증) / 앱 = 수익(리텐션·기록)**

- 웹은 "한 번 쓰고 나가는 계산기"에 강하다 → 검색 유입으로 무료 트래픽.
- 앱은 "개인 데이터를 계속 쌓는 장부"에 강하다 → 리텐션·결제.
- 웹 하단에서 앱으로 내려보내는 **깔때기**를 만든다.

---

## 1. 사이트 구조 (URL)

도메인 후보(가비아/Cloudflare에서 .com 또는 .kr, 연 1~1.5만원):
- `생활계산기` 컨셉: `lifecalc.kr`, `ttok.kr`(똑), `gyesan.kr`(계산)
- SEO상 한글 키워드가 도메인에 없어도 무방 — 콘텐츠/타이틀이 중요.

```
/
├─ /                      포털 홈 (계산기 목록 + 검색)
├─ /lunar                 음력↔양력 변환기          ★ 1순위
├─ /lunar/[date]          예: /lunar/2026-02-17 (공유·색인용 정적 URL)
├─ /gift-money            축의금·조의금 적정액 계산기  ★ 1순위
├─ /memorial-day          제삿날(기일) 계산기         ★ 2순위
├─ /age                   만 나이 계산기 (곁다리, 트래픽 큼)
├─ /saju-day             띠·간지·일진 (음력 엔진 재활용)
└─ /about, /privacy       법적 페이지 (애드센스 승인 필수)
```

**핵심 SEO 트릭:** 변환 결과를 쿼리스트링(`?y=2026`)이 아니라
**정적 경로(`/lunar/2026-02-17`)**로 노출 → 구글이 개별 날짜 페이지를
각각 색인 → "2026년 설날 양력" 같은 롱테일 검색을 통째로 흡수.
(연도·명절 조합으로 수천 개 정적 페이지 자동 생성 = SSG)

## 2. 우선순위 & 검색 수요

| 도구 | 우선순위 | 검색 의도 | 수익 잠재력 | 비고 |
|---|---|---|---|---|
| 음력 변환기 | ★★★ | "음력 변환", "음력 양력" | 中(애드센스) | 코어 자산 직결, 경쟁 약함 |
| 축의금/조의금 계산기 | ★★★ | "축의금 얼마", "조의금 적정액" | **高(제휴)** | 구매 의도 강함 → 화환·답례품 제휴 |
| 제삿날 계산기 | ★★ | "제삿날 계산", "기일 양력" | 中 | 경쟁 거의 없음, 고정 수요 |
| 만 나이 계산기 | ★★ | "만 나이" | 低 | 트래픽 미끼용 (검색량 매우 큼) |
| 띠·일진 | ★ | "내 띠", "오늘 일진" | 低 | 음력 엔진 재활용, 체류시간↑ |

**수익 핵심 통찰:** 애드센스 단가는 낮다. 진짜 돈은 **축의금/조의금 페이지의
구매 의도**에 있다. 조의금을 검색하는 사람은 **지금 장례식장에 갈 사람** →
화환·근조화환·답례품 **제휴 링크(쿠팡파트너스/플라워 제휴)** 전환율이 높다.
→ 트래픽은 음력 변환기로 끌고, **돈은 경조사 페이지에서** 번다.

## 3. 수익 경로 (단계별)

```
Phase A. 트래픽 0~3개월   : 구글 애드센스 (승인 후 자동 광고)
Phase B. 트래픽 검증 후   : 경조사 페이지에 제휴 위젯
         - 근조/축하 화환  → 플라워 제휴 / 쿠팡파트너스
         - 답례품·답례떡    → 쿠팡파트너스
Phase C. 데이터 누적 후   : 앱 다운로드 깔때기 → 경조사 '장부' 앱 결제
```

수익 구조상 **B가 A보다 객단가 10~50배**. A는 사실상 서버비(=0원) 충당용.

## 4. 앱 연동 깔때기 (애플 $99 회수 설계)

웹에서 검증된 도구 중 **"기록·반복"이 필요한 것만** 앱으로 승격:

- 음력 변환기 → 앱으로 안 만든다 (1회성, 웹으로 충분)
- **경조사 장부** → 앱이 정답 (누가 얼마 냈는지 계속 쌓임, 알림, 오프라인)
  - 웹 축의금 계산기 하단: "낸 금액을 기록하고 다음에 돌려받을 때 알림받기 → 앱"
  - 웹이 검색으로 데려온 사람을 앱 설치로 전환

**$99 정당화 로직:** 웹 3종을 6개월 돌려서 트래픽이 도는 게 1~2개 확인되면,
그것들 + 경조사 장부 앱을 **같은 해에 한 계정으로 몰아 출시** → 앱당 비용 $25 이하.

## 5. 기술 스택 (마찰 최소)

| 항목 | 선택 | 이유 |
|---|---|---|
| 프레임워크 | **Astro** (SSG) | 정적 HTML 대량 생성 = SEO 최강, JS 거의 0 → 로딩 빠름 |
| 음력 엔진 | `korean-lunar-calendar` (npm) | KASI/KARI 동일 기준, 검증됨. 추후 KoreanLunarKit로 교체 가능 |
| 호스팅 | **Cloudflare Pages / Vercel** | 무료, git push 자동 배포 |
| 분석 | Cloudflare Web Analytics | 무료, 쿠키리스 |
| 수익 | 애드센스 + 쿠팡파트너스 | 가입만으로 시작 |

**고정비:** 도메인 ~1.5만원/년 **그게 전부.** 서버·빌드·분석 전부 무료 티어.

> iOS 개발자 입장에서 Astro 학습곡선이 부담되면, 1단계는 **단일 HTML + Vanilla JS**로
> 시작해도 된다(이 레포의 `prototype/` 참고). 트래픽이 확인되면 Astro로 승격.

## 6. 실행 순서 (마일스톤)

1. **M1 (1일):** 음력 변환기 단일 페이지 — 입력→결과, 모바일 반응형 ← *지금 단계*
2. **M2 (2~3일):** Astro 프로젝트로 이식 + `/lunar/[date]` 정적 페이지 대량 생성
3. **M3 (1일):** 축의금/조의금 계산기 (룰테이블) + 제휴 위젯 자리
4. **M4 (반나절):** 제삿날 계산기 (음력 엔진 재활용)
5. **M5:** 도메인 연결 + 애드센스 신청 + 구글 서치콘솔 등록
6. **M6 (검증):** 4~8주 트래픽 관찰 → 도는 도구 식별 → 앱 승격 결정

## 7. 검증 기준 (무엇이 "성공"인가)

- 색인 후 8주 내 **월 자연유입(organic) 1,000 세션** 도달한 도구 = 앱 승격 후보.
- 경조사 페이지 제휴 클릭률(CTR) 1% 이상 = 제휴 수익 모델 유효.
- 둘 중 하나라도 충족 → 애플 $99 결제 GO. 둘 다 미달 → 웹 광고만 유지(손실 0).

**리스크 관리:** 최악의 경우에도 손실은 도메인값 1.5만원. 다운사이드가 막혀 있음.
